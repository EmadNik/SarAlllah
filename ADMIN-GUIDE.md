## ۱. معرفی معماری سایت

### ۱.۱  فلسفه طراحی

این سایت **کاملاً ایستا (Static)** است. به این معنا که:

- هیچ سرور application (مانند Node.js، PHP، Python) اجرا نمی‌شود.
- هیچ دیتابیسی وجود ندارد.
- تمام محتوا از فایل‌های فیزیکی در پوشه‌های پروژه خوانده می‌شود.
- تمام منطق در مرورگر کاربر (Client-side) اجرا می‌شود.

این معماری مزایای زیر را دارد:

- **استقرار رایگان** روی GitHub Pages، Netlify، Cloudflare Pages یا هر هاست ایستا.
- **سرعت بالا** — هیچ پردازش سرور不到位.
- **امنیت بالا** — هیچ سطح حمله‌ای وجود ندارد.
- **هزینه صفر** — نیازی به سرور اختصاصی نیست.

### ۱.۲  تکنولوژی‌های استفاده شده

| تکنولوژی | نسخه | نقش |
|----------|------|-----|
| HTML5 | — | ساختار صفحه |
| CSS3 | — | استایل‌دهی (با CSS Variables، Grid، Flexbox) |
| JavaScript (Vanilla) | ES2020+ | منطق برنامه (بدون فریم‌ورک) |
| Google Fonts | — | فونت‌های Vazirmatn و Gulzar |
| Node.js | ۱۸+ | فقط برای اسکریپت `manifest-builder.js` (اختیاری) |

### ۱.۳  چرا Manifest؟

**GitHub Pages قابلیت Directory Listing ندارد.** یعنی اگر کاربر به `https://site.github.io/PIC/` برود، نمی‌تواند لیست فایل‌های آن پوشه را ببیند (خطای ۴۰۴ می‌گیرد). بنابراین سایت نمی‌تواند به‌صورت خودکار فایل‌های یک پوشه را «کشف» کند.

**راه‌حل:** در هر پوشه یک فایل `manifest.json` قرار می‌دهیم که لیست تمام فایل‌های آن پوشه است. JavaScript سایت این manifest را می‌خواند و سپس فایل‌های واقعی را بارگذاری می‌کند.

این کار به دو روش انجام می‌شود:

1. **دستی** — خودتان فایل `manifest.json` را با ویرایشگر متن بسازید.
2. **خودکار (توصیه شده)** — اسکریپت `manifest-builder.js` را اجرا کنید. این اسکریپت تمام پوشه‌ها را اسکن کرده و manifestها را به‌صورت خودکار می‌سازد.

## ۲. ساختار پروژه و فایل‌ها

### ۲.۱  نمای کلی ساختار پوشه‌ها

```
static-site/
│
├── index.html              # صفحه اصلی (نقطه ورود)
├── README.md               # معرفی کوتاه پروژه
├── ADMIN-GUIDE.md          # این فایل (راهنمای جامع مدیر)
├── manifest-builder.js     # اسکریپت ساخت خودکار manifestها
│
├── css/
│   └── styles.css          # تمام استایل‌های سایت
│
├── js/
│   └── app.js              # تمام منطق برنامه
│
├── PIC/                    # 📸 پوشه تصاویر
│   ├── manifest.json       # لیست فایل‌ها (خودکار ساخته می‌شود)
│   ├── image1.jpg
│   ├── image2.png
│   └── ...
│
├── VID/                    # 🎬 پوشه ویدئوها
│   ├── manifest.json
│   ├── video1.mp4
│   └── ...
│
├── MUSIC/                  # 🎵 پوشه مداحی‌ها
│   ├── manifest.json
│   ├── madahi1.mp3
│   └── ...
│
├── SUKH/                   # 🎤 پوشه سخنرانی‌ها
│   ├── manifest.json
│   ├── sokhanrani1.mp3
│   └── ...
│
└── NEW/                    # 📢 پوشه اعلامیه‌ها
    ├── manifest.json
    ├── 1.txt               # شماره ۱ = اعلامیه شاخص
    ├── 2.txt
    └── ...
```

### ۲.۲  فایل‌های مهم و نقش هرکدام

| فایل | نقش | اولویت مدیر |
|------|-----|-------------|
| `index.html` | ساختار صفحه اصلی | پایین (فقط برای تغییرات ساختاری) |
| `css/styles.css` | استایل‌ها | پایین |
| `js/app.js` | منطق برنامه | پایین |
| `manifest-builder.js` | ساخت خودکار manifest | بالا (اجرای منظم) |
| `PIC/manifest.json` | لیست تصاویر | خودکار ساخته می‌شود |
| `VID/manifest.json` | لیست ویدئوها | خودکار ساخته می‌شود |
| `MUSIC/manifest.json` | لیست مداحی‌ها | خودکار ساخته می‌شود |
| `SUKH/manifest.json` | لیست سخنرانی‌ها | خودکار ساخته می‌شود |
| `NEW/manifest.json` | لیست اعلامیه‌ها | خودکار ساخته می‌شود |
| `NEW/*.txt` | فایل‌های اعلامیه | **بسیار بالا** |
| `PIC/*`, `VID/*`, `MUSIC/*`, `SUKH/*` | فایل‌های محتوایی | بالا |

## ۳. سیستم Manifest — چطور کار می‌کند

### ۳.۱  ساختار فایل manifest.json

هر پوشه محتوایی یک فایل `manifest.json` دارد با ساختار زیر:

```json
{
  "files": [
    "file1.jpg",
    "file2.png",
    "file3.webp"
  ]
}
```

### ۳.۲  ساخت خودکار Manifest (توصیه شده)

هر بار که فایلی را به یکی از پوشه‌ها اضافه یا حذف می‌کنید، باید manifestها را به‌روزرسانی کنید. ساده‌ترین راه اجرای اسکریپت `manifest-builder.js` است:

```bash
# در پوشه اصلی پروژه:
node manifest-builder.js
```

این اسکریپت:
- تمام ۵ پوشه (PIC, VID, MUSIC, SUKH, NEW) را اسکن می‌کند.
- فقط فایل‌های با پسوند مجاز را شامل می‌شود.
- فایل `manifest.json` را در هر پوشه بازنویسی می‌کند.

**پیش‌نیاز:** Node.js نسخه ۱۸ یا بالاتر. برای نصب به [nodejs.org](https://nodejs.org) مراجعه کنید.

### ۳.۳  ساخت دستی Manifest

اگر Node.js نصب نیست، می‌توانید manifest را دستی بسازید:

1. در پوشه مورد نظر یک فایل جدید به نام `manifest.json` بسازید.
2. ساختار زیر را وارد کنید و نام فایل‌های خود را جایگزین کنید:

```json
{
  "files": [
    "عاشورا-تصویر-1.svg",
    "عاشورا-تصویر-2.svg"
  ]
}
```

**نکته‌های مهم:**
- نام فایل‌ها باید دقیقاً با نام واقعی فایل (شامل پسوند) مطابقت داشته باشد.
- اگر نام فایل دارای کاراکترهای خاص (فاصله، پرانتز و غیره) است، باید URL-encoded باشد. اما توصیه می‌شود از نام‌های ساده استفاده کنید.
- کاما بعد از آخرین فایل نباید باشد (سینتکس JSON).
- فایل `manifest.json` خود در لیست `files` ظاهر نمی‌شود (اسکریپت آن را فیلتر می‌کند).

### ۳.۴  اگر Manifest مفقود باشد چه می‌شود؟

اگر فایل `manifest.json` در یک پوشه وجود نداشته باشد یا خالی باشد، سایت در آن بخش پیام زیر را نمایش می‌دهد:

> **«هنوز چیزی اضافه نشده است.»**

این رفتار طبیعی است و خطا محسوب نمی‌شود. به محض اینکه فایلی اضافه و manifest ساخته شود، محتوا نمایش داده خواهد شد.

## ۴. راهنمای افزودن محتوا به هر بخش

### ۴.۱  گالری تصاویر (PIC/)

#### قالب‌های مجاز

| قالب | پیشنهاد | توضیح |
|------|---------|-------|
| JPG / JPEG | ✅ بهترین | مناسب برای عکس‌ها |
| WebP | ✅ توصیه شده | حجم کمتر، کیفیت بهتر |
| PNG | ✅ | مناسب برای گرافیک با شفافیت |
| SVG | ✅ | مناسب برای لوگو و آیکون |
| AVIF | ✅ | نسل جدید، حجم بسیار کم |
| GIF | ⚠️ | حجم زیاد، توصیه نمی‌شود |

#### ابعاد و حجم پیشنهادی

- ابعاد پیشنهادی: ۱۲۰۰×۸۰۰ پیکسل یا بیشتر
- حجم حداکثر: **۵۰۰ کیلوبایت** برای هر تصویر
- برای فشرده‌سازی از [TinyPNG](https://tinypng.com) یا [Squoosh](https://squoosh.app) استفاده کنید.

#### مراحل افزودن تصویر جدید

1. فایل تصویر را با نام انگلیسی و بدون فاصله نام‌گذاری کنید (مثلاً `ashura-1.jpg` به‌جای `عاشورا ۱.jpg`).
   - **نکته:** نام فایل به‌عنوان کپشن تصویر استفاده می‌شود. خط تیره (`-`) یا زیرخط (`_`) به فاصله تبدیل می‌شود. مثلاً `ashura_mourning_1.jpg` به‌صورت «ashura mourning 1» نمایش داده می‌شود.
2. فایل را در پوشه `PIC/` کپی کنید.
3. اسکریپت manifest را اجرا کنید: `node manifest-builder.js`
4. تغییرات را commit و push کنید (در فصل ۸ توضیح داده شده).

#### رفتار سایت

- تمام تصاویر در یک شبکه واکنش‌گرا نمایش داده می‌شوند.
- کلیک روی هر تصویر، آن را در **Lightbox** (نمایشگر تمام‌صفحه) باز می‌کند.
- در Lightbox با کلیدهای `←` `→` یا دکمه‌های کناری می‌توانید بین تصاویر جابجا شوید.
- با کلید `ESC` یا کلیک روی ناحیه خارج تصویر، Lightbox بسته می‌شود.

### ۴.۲  گالری ویدئو (VID/)

#### قالب‌های مجاز

| قالب | پیشنهاد | توضیح |
|------|---------|-------|
| MP4 (H.264) | ✅ بهترین | سازگار با تمام مرورگرها |
| WebM (VP9) | ✅ | حجم کمتر، کیفیت بهتر |
| OGG | ⚠️ | پشتیبانی ناقص در Safari |
| MOV | ❌ | باید به MP4 تبدیل شود |
| AVI, MKV | ❌ | باید به MP4 تبدیل شود |

#### محدودیت حجم

- حداکثر حجم هر ویدئو: **۱۰۰ مگابایت**
- برای ویدئوهای طولانی، توصیه می‌شود به بخش‌های ۲۰-۳۰ دقیقه‌ای تقسیم کنید.
- ابزار پیشنهادی برای فشرده‌سازی: [HandBrake](https://handbrake.fr) (رایگان، تنظیم پیش‌فرض "Web" عالی است).

#### مراحل افزودن ویدئو جدید

1. فایل ویدئو را با نام انگلیسی نام‌گذاری کنید (مثلاً `marasem-ashura-1403.mp4`).
2. فایل را در پوشه `VID/` کپی کنید.
3. اسکریپت manifest را اجرا کنید: `node manifest-builder.js`
4. تغییرات را commit و push کنید.

#### رفتار سایت

- ویدئوها به‌صورت کارت با تصویر بندانگشتی (Thumbnail) نمایش داده می‌شوند.
- تصویر بندانگشتی به‌صورت خودکار از ثانیه ۱ ویدئو استخراج می‌شود.
- مدت‌زمان ویدئو به‌صورت خودکار محاسبه و نمایش داده می‌شود.
- کلیک روی کارت، ویدئو را در یک **Modal** (پنجره پاپ‌آپ) باز می‌کند.
- در Modal، کنترل‌های کامل پخش (پخش/توقف، صدا، تمام‌صفحه، جستجو) در دسترس است.
- با کلید `ESC` یا کلیک روی ناحیه خارج، Modal بسته می‌شود.

### ۴.۳  بخش مداحی (MUSIC/)

#### قالب‌های مجاز

| قالب | پیشنهاد | توضیح |
|------|---------|-------|
| MP3 | ✅ بهترین | سازگار با تمام مرورگرها |
| M4A / AAC | ✅ | کیفیت بهتر در حجم کمتر |
| OGG | ⚠️ | پشتیبانی ناقص در Safari |
| WAV | ❌ | حجم بسیار زیاد (تا ۵۰ مگابایت) |
| FLAC | ❌ | حجم زیاد، پشتیبانی ناقص |

#### محدودیت حجم

- حداکثر حجم هر فایل صوتی: **۱۰ مگابایت**
- نرخ بیت پیشنهادی: ۱۲۸ تا ۱۹۲ kbps (تعادل کیفیت و حجم).
- ابزار پیشنهادی برای ویرایش/فشرده‌سازی: [Audacity](https://audacityteam.org) یا [FFmpeg](https://ffmpeg.org).

#### مراحل افزودن مداحی جدید

1. فایل صوتی را با نام انگلیسی نام‌گذاری کنید (مثلاً `salame-hossein.mp3`).
   - نام فایل به‌عنوان عنوان آهنگ نمایش داده می‌شود.
2. فایل را در پوشه `MUSIC/` کپی کنید.
3. اسکریپت manifest را اجرا کنید: `node manifest-builder.js`
4. تغییرات را commit و push کنید.

#### رفتار سایت و پخش‌کننده

- تمام فایل‌های صوتی به‌صورت کارت‌های لیستی نمایش داده می‌شوند.
- هر کارت شامل: عنوان (نام فایل)، قالب، حجم فایل (در صورت امکان)، و مدت‌زمان (به‌صورت خودکار محاسبه می‌شود).
- کلیک روی کارت، آهنگ را در **نوار پخش پایین صفحه (Now Playing Bar)** پخش می‌کند.
- نوار پخش شامل:
  - کاور (با انیمیشن چرخش هنگام پخش)
  - عنوان و نوع (مداحی/سخنرانی)
  - نوار پیشرفت (قابل کلیک برای جستجو در آهنگ)
  - زمان جاری و کل مدت
  - دکمه‌های قبلی/پخش/بعدی
  - دکمه بی‌صدا و کنترل صدا
- **پخش خودکار آهنگ بعدی:** وقتی یک آهنگ تمام می‌شود، آهنگ بعدی در همان لیست به‌طور خودکار پخش می‌شود.
- **لیست پخش مشترک:** بخش مداحی و سخنرانی لیست‌های جداگانه دارند. اگر در حال پخش از لیست مداحی هستید و روی یک سخنرانی کلیک کنید، لیست پخش به سخنرانی تغییر می‌کند.

### ۴.۴  بخش سخنرانی (SUKH/)

این بخش کاملاً مشابه بخش مداحی است، با این تفاوت که:

- فایل‌ها در پوشه `SUKH/` قرار می‌گیرند.
- فقط قالب‌های صوتی مجاز هستند (مانند MUSIC).
- در نوار پخش، نوع «سخنرانی» نمایش داده می‌شود (به‌جای «مداحی»).

تمام مراحل و قالب‌های مجاز با بخش مداحی یکسان است.

---

### ۴.۵  بخش اطلاع‌رسانی (NEW/)

این بخش **مهم‌ترین و حساس‌ترین** بخش برای مدیر سایت است، زیرا فرمت دقیقی دارد.

#### نوع فایل

- فقط فایل‌های `.txt` (متن ساده) مجاز هستند.
- encoding فایل باید **UTF-8** باشد (برای پشتیبانی صحیح از فارسی).

#### نام‌گذاری فایل‌ها (بسیار مهم)

- نام فایل باید با عدد شروع شود: `1.txt`, `2.txt`, `3.txt`, ..., `10.txt`, `11.txt` و غیرا.
- **عدد کوچکتر = اهمیت بیشتر.** فایل `1.txt` به‌عنوان **اعلامیه شاخص** (Featured) در کارت بزرگ نمایش داده می‌شود.
- بقیه فایل‌ها (۲ به بعد) به‌صورت کارت‌های کوچکتر در یک شبکه نمایش داده می‌شوند.
- ترتیب نمایش بر اساس عدد نام فایل است (نه تاریخ ایجاد یا تغییر).

#### قالب محتوای فایل (بسیار مهم)

هر فایل `.txt` باید دقیقاً این قالب را رعایت کند:

```
خط ۱: روز,ماه,سال ساعت:دقیقه
خط ۲: عنوان
خط ۳: سخنران
خط ۴: محل برگزاری
خط ۵ به بعد: توضیحات کامل (می‌تواند چند خط باشد)
```

#### مثال کامل

محتوای فایل `1.txt`:

```
2,مهر,1405 18:23
شام غریبان - روضه خوانی ویژه
حاج محمود کریمی
حسینیه ثارالله، تهران، خیابان ولیعصر
مراسم روضه خوانی ویژه شب شام غریبان به یاد حضرت علی اصغر (ع) و شهدای کربلا.
برنامه شامل اقامه عزای سقا، روضه خوانی، و سینه زنی دسته جمعی خواهد بود.
از همه عزاداران و محبان اهل بیت (ع) دعوت می‌شود تا در این مراسم معنوی شرکت نمایند.
ورود برای عموم آزاد و رایگان است.
```

#### توضیح خط به خط

| خط | محتوا | قالب | مثال |
|----|-------|------|------|
| ۱ | تاریخ و ساعت | `DAY,MONTH,YEAR HOUR:MINUTE` | `2,مهر,1405 18:23` |
| ۲ | عنوان برنامه | متن آزاد | `شام غریبان - روضه خوانی ویژه` |
| ۳ | نام سخنران | متن آزاد | `حاج محمود کریمی` |
| ۴ | محل برگزاری | متن آزاد | `حسینیه ثارالله، تهران` |
| ۵+ | توضیحات | متن آزاد، چند خط | (هرچه می‌خواهید بنویسید) |

#### قواعد دقیق خط ۱ (تاریخ و ساعت)

این خط **بسیار حساس** است. قواعد:

- روز باید عدد باشد (۱ تا ۳۱).
- ماه می‌تواند عدد یا نام فارسی باشد (مهر، آبان، محرم، صفر و غیره).
- سال باید عدد باشد (مثلاً ۱۴۰۵).
- بین ماه و سال باید کاما (`,`) باشد، نه فاصله.
- بین سال و ساعت باید **یک فاصله** باشد.
- ساعت باید با فرمت `HH:MM` باشد (۲۴ ساعته).
- مثال‌های معتبر:
  - `2,مهر,1405 18:23`
  - `15,آبان,1405 20:00`
  - `10,محرم,1446 09:30`
- مثال‌های **نامعتبر**:
  - `2 مهر 1405 18:23` (کاما ندارد)
  - `2,مهر,1405،18:23` (به‌جای فاصله، ویرگول فارسی دارد)
  - `2,مهر,1405 6:5` (دقیقه باید دو رقمی باشد)

#### سناریوهای ویژه

**اگر فایل قالب درست ندارد:**
- آن فایل نادیده گرفته می‌شود و در سایت نمایش داده نمی‌شود.
- در کنسول مرورگر (F12) یک هشدار نمایش داده می‌شود.

**اگر می‌خواهید اعلامیه‌ای را حذف کنید:**
- فایل `.txt` را از پوشه `NEW/` پاک کنید.
- اسکریپت manifest را اجرا کنید.
- commit و push کنید.

**اگر می‌خواهید ترتیب نمایش را تغییر دهید:**
- نام فایل‌ها را تغییر دهید (مثلاً `3.txt` را به `1.txt` تغییر دهید).
- توجه: نمی‌تواند دو فایل هم‌نام داشته باشید. اگر `3.txt` را به `1.txt` تغییر می‌دهید، فایل `1.txt` قبلی را قبلش rename کنید.

**اگر می‌خواهید اعلامیه شاخص را عوض کنید:**
- فایلی که می‌خواهید شاخص شود را به `1.txt` تغییر نام دهید.
- فایل `1.txt` قبلی را به عدد بالاتر تغییر نام دهید.

**اگر خط توضیحات خالی است:**
- خط ۵ به بعد اختیاری است. اگر خالی باشد، فقط کارت بدون توضیحات نمایش داده می‌شود.
- اما خطوط ۱ تا ۴ **الزامی** هستند.

## ۵. قواعد دقیق فایل‌های اعلامیه

### ۵.۱  خلاصه قواعد

| قاعده | توضیح |
|-------|-------|
| پسوند فایل | فقط `.txt` |
| Encoding | UTF-8 |
| نام فایل | باید با عدد شروع شود (`1.txt`, `2.txt`, ...) |
| تعداد خطوط | حداقل ۴ خط (خطوط ۱ تا ۴ الزامی هستند) |
| فرمت خط ۱ | `DAY,MONTH,YEAR HOUR:MINUTE` |
| فاصله بین سال و ساعت | دقیقاً یک فاصله |
| جداکننده بین روز/ماه/سال | کاما انگلیسی (`,`) نه فارسی (`،`) |
| ساعت | فرمت ۲۴ ساعته `HH:MM` (دقیقه دو رقمی) |
| خطوط ۵ به بعد | توضیحات، می‌تواند چند خطی باشد |

### ۵.۲  اولویت و ترتیب نمایش

- فایل با کوچک‌ترین عدد → **اعلامیه شاخص** (Featured) — کارت بزرگ بالا.
- بقیه فایل‌ها → کارت‌های کوچک در شبکه زیر کارت شاخص.
- ترتیب کارت‌های کوچک نیز بر اساس عدد نام فایل است (صعودی).

### ۵.۳  سناریوی چندگانه

| موقعیت | اقدام |
|--------|-------|
| افزودن اعلامیه جدید در انتهای لیست | آخرین عدد + ۱ را برای نام فایل استفاده کنید |
| افزودن اعلامیه جدید به‌عنوان شاخص | فایل `1.txt` فعلی را به عدد بالاتر تغییر نام دهید، سپس اعلامیه جدید را با نام `1.txt` ذخیره کنید |
| حذف اعلامیه شاخص | فایل `1.txt` را پاک کنید. اعلامیه `2.txt` به‌طور خودکار شاخص جدید می‌شود (اما نام فایل تغییر نمی‌کند) |
| جابجایی ترتیب | نام فایل‌ها را swap کنید (مثلاً `3.txt` ↔ `1.txt` با نام موقت) |
| اعلامیه موقت (مثلاً برای یک هفته) | فایل را با عدد بالا اضافه کنید (مثلاً `99.txt`). بعد از انقضا، فایل را پاک کنید. |

## ۶. ترتیب بخش‌های سایت

ترتیب نمایش بخش‌ها در صفحه اصلی (از بالا به پایین):

1. **هیرو (Hero)** — تیتر اصلی سایت با گرادیان دراماتیک
2. **درباره هیئت (About)** — معرفی رسالت و آمار هیئت
3. **فعالیت‌ها (Activities)** — چهار کارت فعالیت‌های هیئت (روضه‌خوانی، سینه‌زنی، سخنرانی، پذیرایی)
4. **گالری رسانه (Media)** — بخش یکپارچه شامل تصاویر، ویدئوها، مداحی‌ها و سخنرانی‌ها با فیلتر دسته‌بندی
5. **لوکیشن (Location)** — نقشه تعاملی، نشانی، تلفن، ایمیل، ساعات کاری و دکمه مسیریابی
6. **اطلاع‌رسانی (Announcements)** — اعلامیه شاخص + کارت‌های کوچک (`/NEW`)
7. **ارتباط با ما (Contact)** — دو ستونه: اطلاعات تماس (چپ) + فرم غیرفعال (راست) با پیام «ارسال پیام در حال حاضر ممکن نیست»
8. **فوتر** — لینک‌های سریع، شبکه‌های اجتماعی، دکمه بازگشت به بالا و کپی‌رایت «© ۱۴۰۵»

### ۶.۱  تغییر ترتیب بخش‌ها

برای تغییر ترتیب، باید فایل `index.html` را ویرایش کنید. هر بخش یک `<section>` با `id` مشخص است:

- `<section id="hero">` — هیرو
- `<section id="about">` — درباره هیئت
- `<section id="activities">` — فعالیت‌ها
- `<section id="media">` — گالری رسانه یکپارچه
- `<section id="location">` — لوکیشن
- `<section id="announcements">` — اطلاع‌رسانی
- `<section id="contact">` — ارتباط با ما

برای جابجایی، کل بلاک `<section>...</section>` را cut کرده و در جای جدید paste کنید. توصیه می‌شود این کار را فقط در صورت نیاز انجام دهید و حتماً لینک‌های ناوبری (navbar) را نیز به‌روزرسانی کنید.

### ۶.۲  گالری رسانه یکپارچه

از نسخه جدید، تمام رسانه‌ها (تصاویر، ویدئوها، مداحی‌ها، سخنرانی‌ها) در یک بخش واحد با نام **«گالری رسانه»** نمایش داده می‌شوند. کاربر می‌تواند با استفاده از فیلترهای بالای گرید، تنها یک نوع رسانه را ببیند:

- **همه** — نمایش تمام رسانه‌ها
- **تصاویر** — فقط تصاویر از پوشه `/PIC`
- **ویدئو** — فقط ویدئوها از پوشه `/VID`
- **مداحی** — فقط مداحی‌ها از پوشه `/MUSIC`
- **سخنرانی** — فقط سخنرانی‌ها از پوشه `/SUKH`

با کلیک روی هر کارت، یک **Lightbox** زیبا باز می‌شود که رسانه را در اندازه کامل نمایش می‌دهد. در Lightbox می‌توانید با کلیدهای `←` `→` یا دکمه‌های کناری بین رسانه‌ها جابجا شوید و با `ESC` ببندید.

## ۷. به‌روزرسانی سایت

### ۷.۱  فلو کاری استاندارد

برای افزودن یا تغییر محتوا، همیشه این ۴ مرحله را دنبال کنید:

#### مرحله ۱: افزودن یا حذف فایل‌ها

فایل‌های جدید را در پوشه مربوطه کپی کنید (`PIC/`, `VID/`, `MUSIC/`, `SUKH/`, `NEW/`). برای حذف، فایل‌ها را از پوشه پاک کنید.

#### مرحله ۲: ساخت Manifest

```bash
node manifest-builder.js
```

این کار `manifest.json` را در تمام پوشه‌ها به‌روزرسانی می‌کند.

#### مرحله ۳: Commit و Push

```bash
git add .
git commit -m "Add new content: 3 images, 1 announcement"
git push origin main
```

#### مرحله ۴: انتظار برای انتشار

- GitHub Pages معمولاً ۱ تا ۵ دقیقه طول می‌کشد تا تغییرات را منتشر کند.
- در تنظیمات repository به **Settings → Pages** بروید تا وضعیت build را ببینید.

### ۷.۲  بدون Node.js

اگر Node.js نصب نیست:

1. فایل `manifest.json` را در پوشه مورد نظر با ویرایشگر متن باز کنید.
2. نام فایل‌های جدید را به آرایه `files` اضافه کنید (یا فایل‌های حذف شده را پاک کنید).
3. فایل را ذخیره کنید.
4. Commit و push کنید.

### ۷.۳  ویرایش اطلاعات تماس و آدرس

در نسخه فعلی، اطلاعات تماس در فایل `index.html` هاردکد شده است. برای تغییر:

1. فایل `index.html` را باز کنید.
2. بخش `<section id="contact">` را پیدا کنید.
3. متن داخل `<p>` ها را ویرایش کنید.
4. ذخیره، commit و push کنید.

## ۸. راهنمای استقرار روی GitHub Pages

### ۸.۱  پیش‌نیازها

- یک حساب GitHub (رایگان)
- نصب Git روی سیستم خود ([git-scm.com](https://git-scm.com))

### ۸.۲  مراحل استقرار

#### مرحله ۱: ساخت Repository جدید

1. به [github.com](https://github.com) وارد شوید.
2. روی `+` در گوشه بالا کلیک کنید → **New repository**.
3. نام repository را وارد کنید (مثلاً `tharullah`).
4. تنظیمات:
   - Public (توصیه شده)
   - **Initialize this repository with a README** را **فعال نکنید** (ما فایل‌ها را از سیستم خود push می‌کنیم).
5. روی **Create repository** کلیک کنید.

#### مرحله ۲: آماده‌سازی فایل‌ها

تمام فایل‌های پروژه (پوشه `static-site/`) را در یک پوشه محلی روی سیستم خود قرار دهید. سپس در ترمینال (یا Command Prompt) به آن پوشه بروید:

```bash
cd /path/to/static-site
```

#### مرحله ۳: Init و Commit اولیه

```bash
git init
git add .
git commit -m "Initial commit: Tharullah static website"
git branch -M main
```

#### مرحله ۴: اتصال به GitHub

دستورات زیر را اجرا کنید (به‌جای `USERNAME` و `REPO-NAME` مقادیر خود را جایگزین کنید):

```bash
git remote add origin https://github.com/USERNAME/REPO-NAME.git
git push -u origin main
```

#### مرحله ۵: فعال‌سازی GitHub Pages

1. در صفحه GitHub repository خود، به **Settings** بروید.
2. از منوی سمت چپ، **Pages** را انتخاب کنید.
3. در بخش **Build and deployment → Source**، گزینه **Deploy from a branch** را انتخاب کنید.
4. در بخش **Branch**:
   - شاخه: `main`
   - پوشه: `/ (root)`
5. روی **Save** کلیک کنید.

#### مرحله ۶: دریافت آدرس سایت

پس از چند دقیقه، سایت شما در آدرس زیر در دسترس خواهد بود:

```
https://USERNAME.github.io/REPO-NAME/
```

مثلاً اگر نام کاربری شما `ali-hosseini` و نام repository `tharullah` است:

```
https://ali-hosseini.github.io/tharullah/
```

### ۸.۳  استفاده از دامنه اختصاصی (اختیاری)

اگر دامنه اختصاصی دارید (مثلاً `tharullah.ir`):

1. در تنظیمات DNS دامنه خود، یک رکورد CNAME با مقدار `USERNAME.github.io` اضافه کنید.
2. در GitHub به **Settings → Pages → Custom domain** بروید.
3. دامنه خود را وارد و **Save** کنید.
4. منتظر بمانید تا گزینه **Enforce HTTPS** فعال شود (چند ساعت).

### ۸.۴  آپدیت‌های بعدی

هر بار که تغییری ایجاد کردید، فقط کافی است:

```bash
git add .
git commit -m "Update content"
git push origin main
```

GitHub Pages به‌صورت خودکار تغییرات را در چند دقیقه منتشر می‌کند.

## ۹. مدیریت پیشرفته و نگهداری

### ۹.۱  بکاپ‌گیری

توصیه می‌شود هفتگی از کل پروژه بکاپ بگیرید:

```bash
# ایجاد آرشیو zip از کل پروژه
zip -r tharullah-backup-$(date +%Y%m%d).zip static-site/
```

این فایل را روی یک سرویس ابری (Google Drive، Dropbox و غیره) ذخیره کنید.

### ۹.۲  بهینه‌سازی عملکرد

- **فشرده‌سازی تصاویر:** همیشه قبل از آپلود، تصاویر را با TinyPNG فشرده کنید.
- **فشرده‌سازی ویدئو:** همیشه قبل از آپلود، ویدئوها را با HandBrake فشرده کنید.
- **نام‌گذاری صحیح:** از نام‌های کوتاه و انگلیسی استفاده کنید تا URLها تمیز بمانند.
- **حذف فایل‌های قدیمی:** هر ۳ ماه یک‌بار، فایل‌های قدیمی را که دیگر لازم نیستند، حذف کنید.

### ۹.۳  سئو (SEO)

این سایت به‌صورت پیش‌فرض دارای متاتگ‌های مناسب سئو است. برای بهبود بیشتر:

- در فایل `index.html`، تگ `<title>` و `<meta name="description">` را به‌روزرسانی کنید.
- اگر دامنه اختصاصی دارید، آن را در [Google Search Console](https://search.google.com/search-console) ثبت کنید.

### ۹.۴  امنیت

- هیچ‌گاه اطلاعات حساس (پسورد، کلید API) را در فایل‌های پروژه قرار ندهید.
- GitHub Pages به‌صورت خودکار HTTPS را فعال می‌کند — نیازی به کار اضافی نیست.
- اگر کاربران دیگری به repository دسترسی دارند، از Branch Protection Rules استفاده کنید.

### ۹.۵  همکاری تیمی

اگر چند نفر روی سایت کار می‌کنند:

1. در GitHub به **Settings → Manage access** بروید.
2. همکاران را دعوت کنید.
3. هر همکار باید تغییری را در یک branch جداگانه (مثلاً `feature/new-announcement`) ایجاد کند.
4. تغییرات از طریق Pull Request ادغام شوند.

## ۱۰. عیب‌یابی

### ۱۰.۱  مشکل: پیام «هنوز چیزی اضافه نشده است» نمایش داده می‌شود

**علت:** فایل `manifest.json` در پوشه مورد نظر وجود ندارد، خالی است، یا ساختار JSON آن نامعتبر است.

**راه‌حل:**
1. مطمئن شوید فایل‌ها واقعاً در پوشه قرار دارند.
2. اسکریپت `node manifest-builder.js` را اجرا کنید.
3. اگر manifest را دستی ساخته‌اید، سینتکس JSON را با یک اعتبارسنج (مانند jsonlint.com) بررسی کنید.
4. commit و push کنید و چند دقیقه صبر کنید.

### ۱۰.۲  مشکل: فایل پخش نمی‌شود

**علت:** لینک فایل اشتباه است، یا فایل از سرور قابل دسترسی نیست.

**راه‌حل:**
1. آدرس مستقیم فایل را در مرورگر امتحان کنید (مثلاً `https://USERNAME.github.io/REPO-NAME/MUSIC/song.mp3`).
2. اگر خطای ۴۰۴ می‌گیرید، یعنی نام فایل در manifest با نام واقعی فایل مطابقت ندارد.
3. اگر فایل دانلود می‌شود اما پخش نمی‌شود، قالب فایل را بررسی کنید (باید MP3, M4A یا OGG باشد).
4. کنسول مرورگر (F12) را برای خطاهای CORS یا network بررسی کنید.

### ۱۰.۳  مشکل: اعلامیه نمایش داده نمی‌شود

**علت:** فرمت فایل `.txt` اشتباه است.

**راه‌حل:**
1. فایل را با ویرایشگر متن باز کنید.
2. مطمئن شوید خط ۱ دقیقاً با قالب `DAY,MONTH,YEAR HOUR:MINUTE` مطابقت دارد.
3. مطمئن شوید خطوط ۲، ۳، ۴ وجود دارند (عنوان، سخنران، محل).
4. encoding فایل را UTF-8 قرار دهید.
5. در Notepad Windows: **Save As → Encoding: UTF-8**.
6. کنسول مرورگر را برای هشدارهای parsing بررسی کنید.

### ۱۰.۴  مشکل: تغییرات در سایت نمایش داده نمی‌شود

**علت:** کش مرورگر، یا تأخیر GitHub Pages.

**راه‌حل:**
1. صفحه را با `Ctrl + F5` (یا `Cmd + Shift + R` در مک) سخت رفرش کنید.
2. در حالت ناشناس (Incognito) سایت را باز کنید.
3. ۵-۱۰ دقیقه صبر کنید (GitHub Pages ممکن است تأخیر داشته باشد).
4. در GitHub به **Settings → Pages** بروید و وضعیت build را بررسی کنید.

### ۱۰.۵  مشکل: نام‌های فارسی به‌هم‌ریخته نمایش داده می‌شوند

**علت:** encoding فایل UTF-8 نیست، یا نام فایل دارای کاراکترهای غیرمجاز است.

**راه‌حل:**
1. مطمئن شوید encoding فایل `.txt` UTF-8 است.
2. برای نام فایل‌ها از حروف انگلیسی و اعداد استفاده کنید.
3. اگر مجبور به استفاده از فارسی هستید، مطمئن شوید URL-encoded در manifest قرار دارند (اما توصیه نمی‌شود).

### ۱۰.۶  مشکل: اسکریپت manifest-builder.js اجرا نمی‌شود

**علت:** Node.js نصب نیست یا نسخه قدیمی است.

**راه‌حل:**
1. از [nodejs.org](https://nodejs.org) Node.js نسخه ۱۸ یا بالاتر را نصب کنید.
2. در ترمینال `node --version` را اجرا کنید تا نصب بودن تأیید شود.
3. اگر Node نصب نیست، manifest را دستی بسازید (در فصل ۳ توضیح داده شده).

## ۱۱. سوالات متداول

**س: آیا می‌توانم از هاست دیگری به‌جز GitHub Pages استفاده کنم؟**
بله. این سایت کاملاً ایستا است و روی هر هاست ایستا (Netlify، Cloudflare Pages، Vercel، یا حتی هاست cPanel معمولی) کار می‌کند. فقط فایل‌ها را آپلود کنید.

**س: چطور می‌توانم لوگو یا نام سایت را تغییر دهم؟**
در فایل `index.html`، تگ `<span class="title-calligraphy">هیئت ثارالله</span>` را پیدا و ویرایش کنید.

**س: آیا می‌توانم فونت سایت را تغییر دهم؟**
بله. در فایل `css/styles.css`، در بخش `:root`، متغیر `--font-body` و `--font-calligraphy` را تغییر دهید. اگر از Google Fonts استفاده می‌کنید، لینک آن را در `<head>` فایل `index.html` اضافه کنید.

**س: چطور می‌توانم رنگ‌های سایت را تغییر دهم؟**
در فایل `css/styles.css`، در بخش `:root`، تمام رنگ‌ها در متغیرهای CSS تعریف شده‌اند. کافی است مقادیر را تغییر دهید.

**س: چطور می‌توانم بخش پخش زنده را فعال کنم؟**
در نسخه فعلی، بخش پخش زنده همیشه غیرفعال است. برای فعال‌سازی، باید یک سرویس استریم (YouTube Live، آپارات، یا سرویس RTMP) راه‌اندازی کنید و سپس در فایل `index.html` و `js/app.js` تغییرات لازم را اعمال کنید. این یک توسعه پیشرفته است.

**س: چطور می‌توانم فرم تماس را فعال کنم؟**
در نسخه فعلی، فرم تماس غیرفعال است. برای فعال‌سازی، می‌توانید از سرویس‌های رایگان مانند [Formspree](https://formspree.io) یا [FormSubmit](https://formsubmit.co) استفاده کنید که فرم‌های ایستا را به ایمیل تبدیل می‌کنند.

**س: آیا می‌توانم چندین announcement شاخص داشته باشم؟**
خیر. در حال حاضر فقط فایل `1.txt` به‌عنوان شاخص نمایش داده می‌شود. اگر می‌خواهید چند اعلامیه مهم را برجسته کنید، می‌توانید به‌صورت دستی در فایل `index.html` کارت‌های اضافی اضافه کنید.

**س: چه اندازه‌ای برای تصاویر گالری بهتر است؟**
ترکیبی از تصاویر مربعی (۱:۱) و افقی (۱۶:۹ یا ۳:۲) بهترین نتیجه را در گرید ماسونی می‌دهد. حداقل ابعاد: ۸۰۰×۶۰۰ پیکسل.

**س: آیا می‌توانم فایل‌های با نام فارسی اضافه کنم؟**
از نظر فنی بله، اما توصیه نمی‌شود. نام‌های فارسی ممکن است در URLencoding به مشکلاتی منجر شوند. همیشه از نام‌های انگلیسی استفاده کنید. برای نمایش نام فارسی به کاربر، می‌توانید نام فایل را انگلیسی اما با الگوی مشخص انتخاب کنید (مثلاً `madahi-salame-hossein.mp3` که به‌صورت «madahi salame hossein» نمایش داده می‌شود).

**س: اگر فایل‌های زیادی (مثلاً ۱۰۰ فایل) در یک پوشه داشته باشم چه؟**
سایت به‌خوبی با ۱۰۰ فایل کار می‌کند. اما برای عملکرد بهتر:
- تصاویر را فشرده کنید (زیر ۲۰۰ کیلوبایت).
- ویدئوها را به بخش‌های کوچک‌تر تقسیم کنید.
- از lazy loading استفاده می‌شود (به‌صورت خودکار در سایت فعال است).

### ۱۲.۳  لیست بررسی نهایی قبل از هر آپدیت

قبل از commit و push، این موارد را بررسی کنید:

- [ ] اسکریپت `node manifest-builder.js` اجرا شده است
- [ ] در مرورگر محلی تست شده است (با `python3 -m http.server 8765`)
- [ ] commit message واضح و توصیفی نوشته شده است
